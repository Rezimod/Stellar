# Stellar — Design Context

## Design Context

### Users

**Primary user of every Stellar page: an Astroman telescope buyer asking "can I use my new scope tonight?"**

They are not a prediction market degen. They are not a crypto native. They are a person who spent $800 on a Dobsonian and wants to aim it at something worthwhile. The prediction markets are a side activity that rewards the observing they already do — not the other way around.

The `/sky` page specifically is opened across three contexts, all of which must work:

1. **Daytime planning (desk, coffee)** — "is tonight worth setting up for?" Reading-focused, full detail OK.
2. **Dusk go/no-go (phone, kitchen)** — "next 30 min, yes or no?" One-glance verdict.
3. **11pm in the field (phone, cold fingers, red night-vision mode welcome)** — "where is Saturn right now?" High-contrast, large tap targets, no unnecessary motion.

Secondary users: Stars holders hunting observer-advantage missions; tourists/casuals with no gear checking "what's up tonight?"

### Brand Personality

**Patient. Precise. Earned.**

- **Patient** — astronomy rewards waiting. Skies clear, planets rise, comets return. The interface should never feel hurried. No countdown timers. No "limited time." No trending-ticker theater.
- **Precise** — every number is measured by a real instrument (Open-Meteo, NOAA, IMO, astronomy-engine). We are not guessing. Display precision at the level the instrument actually provides — no fake decimals, no rounded-up confidence.
- **Earned** — the edge in this product comes from going outside. Rewards are not promotional; they are paid for with cold fingers and clear eyes. The design should look like something you'd be given by a guild, not sold by a growth team.

Emotional goals: **confidence** (the number is right), **calm** (you don't have to rush), **reverence** (the sky is older than the app).

### Aesthetic Direction

**Reference: nasa.gov** — dense information presented with restraint. Rectangular compositions. Real photography of real space, not stylized vector art. Monospace for numbers, not for "dev vibes." No animation theater. Type does the work.

**Anti-reference: Kalshi / Polymarket** — gambling skin over a serious product. Rounded pastel pills, confetti percentages, green/red urgency, "trending markets" ticker. Every pattern that says "quick money" is wrong for us. We also avoid the generic shadcn dashboard look and the cyan-on-dark / purple-gradient AI-slop palette.

**Theme: dark, but not black.** Primary viewing context is dusk and night; astronomers protect their dark adaptation. Use a tinted near-black around `oklch(0.15 0.015 260)` — a night-sky color, not VSCode's. Ship a companion light theme for the daytime-planning context; derive both from the same OKLCH ramp so values are perceptually parallel.

**Palette discipline:**
- Surface: 60%, tinted indigo `oklch(0.15 0.015 260)` → `oklch(0.18 0.015 260)`
- Text & structure: 30%, warm off-white `oklch(0.94 0.015 80)` (light type gets +0.05 line-height for breathing room on dark)
- Accent: 10%, one color at a time — either **Stars gold** `oklch(0.82 0.15 80)` for Stars/rewards OR **verdict green/amber/red** for sky state. Never both simultaneously in the same viewport.
- No violet glows. No purple-to-blue gradients. No gradient text.

**Typography (currently shipped — wired through `--font-display` / `--font-body` / `--font-mono` in `src/app/globals.css`):**
- **Headings** (`h1`–`h6`) — `Geist` **Medium (500)**. `globals.css` overrides headings to `--font-body`, so Geist is the shipped heading voice (source of truth). `Orbitron` (`--font-display`) is kept as a **deliberate display accent** — the verdict label and select hero type via the `.font-display` utility — not a blanket heading face. Geometric, technical; use sparingly, never as body.
- **Body / paragraphs / UI copy** — `Geist` **Medium (500)**. Clean modern sans, legible at small sizes. Wired via `--font-body` (and aliased to `--font-cta`). Use the `.font-body` utility or the var directly — never hardcode a font name.
- **Numbers / coordinates / times / mono** — `JetBrains Mono`. Tabular numerals `font-variant-numeric: tabular-nums`. Used for every number that must align vertically: cloud cover, humidity, altitudes, RA/Dec, timestamps. Wired via `--font-mono`.
- **Georgian (KA locale)** — `Noto Sans Georgian`. Pairs cleanly with Geist and inherits the Geist weight.
- **Banned font names — never hardcode any of these in new components:** `Inter`, `Saira`, `Source Serif 4`, `Public Sans`, `Cormorant Garamond`, `Fraunces`, `Space Grotesk`. Always go through the token vars or the `.font-display` / `.font-body` utilities.

**What we are removing from the current UI:**
- Glass-card-in-glass-card nesting (every section wraps in `glass-card`)
- Violet glow shadows on hover
- Gradient accent on the score ring fill
- 9–11px microcopy crammed to look "technical" — floor at 12px, use hierarchy not size-compression
- `border-left: 3px solid ...` patterns and `background-clip: text` gradients if any survive

### Design Principles

1. **Verdict first, detail on demand.** Every page answers one primary question in under a second. Supporting data is present but quiet, revealed through scroll and interaction, never competing with the verdict.
2. **Numbers earn their position.** A stat is visible only when it changes a decision. Decorative stats (the 2×2 grid of everything-we-measure) get replaced by a single inline sentence that highlights what's actually notable tonight.
3. **No glass, no glow, no gradients.** Strip the decorative layer. Typography, spacing, and real photography carry the design. If a border is needed, one hairline at `oklch(0.94 0.015 80 / 0.08)` is enough.
4. **Photograph where you can, illustrate where you must.** Real moon phases (SVG derived from live illumination), real planetary imagery, real deep-sky targets. Stylized vector "space icons" are a generic-AI tell — avoid.
5. **Mono for data, sans for voice, serif only for the verdict.** Three roles, three families. Never a serif label. Never mono for "this is a developer product." The reader should be able to feel which role each word is playing.

### Accessibility

- WCAG 2.1 AA minimum across both themes. Text contrast ≥ 4.5:1; UI ≥ 3:1. Verdict colors (go/maybe/skip) must never carry meaning alone — pair with icon and label.
- Respect `prefers-reduced-motion` globally. Page-load entrance is the only choreographed moment; everything else is instant.
- Red-light / dark-adaptation friendly mode as a future goal: an optional deep-red theme for observers actually at the eyepiece. Not shipped yet, but the token system should make it a one-file addition.
- Bilingual typography: Geist pairs cleanly with Noto Sans Georgian — keep both loaded for EN + KA.
